### Предложения по улучшению продукта

Архитектурные
Внедрить выделенный Inventory API с эндпоинтами:

POST /inventory/reserve — резервирование товара (с orderId, productId, quantity)

POST /inventory/release — возврат товара при отмене заказа

GET /inventory/check — проверка наличия

Добавить поле orderId в модель резервирования для связи с заказом

Реализовать транзакционность — создание заказа и резервирование в одной транзакции (или SAGA-паттерн)

Внедрить оптимистические блокировки (version field) для защиты от race condition

### UI

Добавить проверку наличия товара перед отправкой формы (вызов /inventory/check)

Показывать спиннер на кнопке отправки во время загрузки

Улучшить сообщения об ошибках:

«Товара нет в наличии» вместо «Ошибка сервера»

«Не удалось зарезервировать товар, попробуйте позже»

### API

quantity должна быть положительным целым числом

Добавить эндпоинт для частичного обновления количества (PATCH /notes/{id}/quantity)
