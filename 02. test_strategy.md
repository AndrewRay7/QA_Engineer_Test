## Стратегия тестирования

### Scope (что тестируем)

UI форма заказа — функциональность, валидация, UX, сетевые запросы

API склада (Notes API) — аутентификация, CRUD, негативные сценарии

Интеграция — сценарии взаимодействия заказов и склада (насколько позволяет текущая реализация)

### Out-of-Scope (не тестируем)

Нагрузочное тестирование

Тестирование производительности

Безопасность (кроме базовых XSS/SQL-инъекций)

Тестирование мобильных устройств

### Подход

Исследовательское тестирование — для UI формы

API-тестирование — через Swagger/Postman

Анализ расхождений — сравнение требований с реальным поведением
